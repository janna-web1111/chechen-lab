## 1. Назначение документа

## 2. Общие принципы интерфейса

Интерфейс должен быть:

- мобильным в первую очередь;
- простым для ребенка и взрослого начинающего;
- визуально спокойным;
- понятным без длинных инструкций;
- удобным для коротких занятий;
- доступным для повторного возвращения.

## 3. Основные экраны MVP

### Стартовый экран

Назначение: быстро объяснить пользователю, что это приложение для изучения чеченского языка, и дать начать обучение.

Элементы:

- название проекта;
- краткий смысл;
- кнопка начала обучения;
- отображение пути A0 -> A1.

### Экран пути обучения

Назначение: показать прогресс по уровню A0 -> A1.

Элементы:

- список тем;
- статус каждой темы;
- текущая тема;
- общий прогресс.

### Экран темы

Назначение: показать содержание одной темы.

Элементы:

- название темы;
- короткое описание;
- уроки или карточки;
- кнопка запуска квиза;
- прогресс темы.

### Экран урока

Назначение: провести пользователя через короткий учебный блок.

Элементы:

- учебный текст;
- карточки;
- аудио-кнопка;
- кнопки перехода вперед и назад;
- индикатор шага.

### Экран карточки

Назначение: помочь запомнить слово или фразу.

Элементы:

- чеченское слово или фраза;
- перевод;
- аудио-кнопка;
- транскрипция или подсказка, если нужна;
- кнопка следующей карточки.

### Экран квиза

Назначение: проверить понимание темы.

Элементы:

- вопрос;
- варианты ответа;
- состояние выбранного ответа;
- обратная связь;
- прогресс по вопросам.

### Экран результата

Назначение: показать итог квиза или темы.

Элементы:

- количество правильных ответов;
- процент результата;
- рекомендация повторить или продолжить;
- кнопка перехода дальше.

### Экран повторения

Назначение: дать пользователю повторить изученные слова.

Элементы:

- список карточек;
- фильтр по теме;
- аудио;
- статус повторения.

## 4. Состояния интерфейса

Для каждого ключевого экрана нужно предусмотреть:

- загрузку;
- пустое состояние;
- ошибку;
- отсутствие аудио;
- отсутствие сохраненного прогресса;
- успешное завершение действия.

## 5. Требования к текстам интерфейса

Тексты должны быть:

- короткими;
- понятными;
- без сложной терминологии;
- на русском языке в MVP;
- готовыми к переводу на другие языки в будущем.

## 6. Вопросы для уточнения

- Нужно ли использовать кириллицу или латиницу для чеченских слов в MVP?
- Нужна ли темная тема?
- Должен ли интерфейс быть одинаковым для детей и взрослых?
- Нужна ли отдельная главная страница или сразу экран обучения?
- Какие визуальные элементы лучше подходят для детской аудитории?
