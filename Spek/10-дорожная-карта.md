# Дорожная карта

## 1. Назначение документа

Этот документ описывает план развития проекта по этапам: что нужно сделать сначала, что можно добавить после MVP и в каком направлении должен развиваться продукт.

Документ связан с:

- [01-бизнес-контекст.md](01-бизнес-контекст.md);
- [06-спецификация-ai-логики.md](06-спецификация-ai-логики.md);
- [08-техническая-спецификация.md](08-техническая-спецификация.md);
- [09-спецификация-mvp.md](09-спецификация-mvp.md).

## 2. Этап 1: Подготовка спецификаций

Цель: зафиксировать логику проекта до разработки.

Результат:

- бизнес-контекст;
- спецификация пользователей;
- функциональная спецификация;
- спецификация данных;
- спецификация процессов;
- спецификация интерфейса;
- техническая спецификация;
- спецификация MVP.

## 3. Этап 2: Подготовка контента для MVP

Цель: подготовить учебный материал для первого пути A0 -> A1.

Результат:

- 5 первых тем;
- карточки для каждой темы;
- переводы;
- аудио;
- квизы;
- проверка чеченского контента.

## 4. Этап 3: Разработка MVP

Цель: создать первую рабочую версию приложения.

Результат:

- мобильный интерфейс;
- путь A0 -> A1;
- карточки;
- аудио;
- квизы;
- сохранение прогресса;
- базовое повторение.

## 5. Этап 4: Проверка MVP

Цель: понять, нужен ли продукт пользователям и что нужно улучшить.

Проверяется:

- проходит ли пользователь первый урок;
- завершает ли первую тему;
- возвращается ли позже;
- понятен ли интерфейс;
- хватает ли аудио;
- интересны ли карточки и квизы;
- полезен ли продукт родителям и детям.

## 6. Этап 5: Улучшения после MVP

Возможные улучшения:

- больше тем A1;
- уровень A2;
- повторение по ошибкам;
- streak;
- личные кабинеты;
- родительский обзор прогресса;
- PWA;
- офлайн-режим;
- улучшенная аналитика;
- админ-инструменты для контента.

## 7. Этап 6: Мультиязычность

После проверки MVP можно добавлять языки интерфейса.

Приоритетные языки:

- английский;
- немецкий;
- французский;
- нидерландский;
- польский.

## 8. Этап 7: Расширение уровней

После A0 -> A1 можно развивать:

- A1 -> A2;
- A2 -> B1;
- B1 -> B2.

Каждый новый уровень должен иметь отдельный набор тем, карточек, аудио, квизов и критериев прогресса.

## 9. Вопросы для уточнения

- Какой срок нужен для подготовки первого контента?
- Кто будет проверять чеченский язык и аудио?

- Какие улучшения должны идти сразу после MVP?
- Какие языки интерфейса важнее по реальной аудитории?
