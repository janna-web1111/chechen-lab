# Чеклист релиза MVP

## Основной пользовательский путь

- Новый пользователь открывает приложение.
- Пользователь видит понятный старт.
- Пользователь переходит к пути A0 -> A1.
- Пользователь видит 5 тем.
- Пользователь открывает тему.
- Пользователь проходит карточки.
- Пользователь проходит квиз.
- Пользователь видит результат.
- Прогресс сохраняется после перезагрузки страницы.
- Пользователь может вернуться к изученным словам.

## Данные и контент

- Контент отделен от UI-компонентов.
- У каждой темы есть ID, порядок, карточки и квиз.
- У карточек есть чеченский текст, русский перевод и статус проверки.
- Непроверенный чеченский контент не показан как финальный.
- Квиз не содержит одинаковых вариантов ответа.
- Порог результата соответствует спецификации или явно задан в данных.

## UI и состояния

- Есть состояние нового пользователя без прогресса.
- Есть восстановление сохраненного прогресса.
- Есть пустое состояние, если контент не загрузился или отсутствует.
- Есть понятная ошибка при проблеме данных.
- Тексты помещаются в элементы интерфейса на мобильном размере.
- Кнопки и интерактивные элементы доступны с клавиатуры, если это применимо.

## Техника

- Прогресс хранится в `localStorage` с namespace.
- Не сохраняются персональные данные ребенка или семьи.
- Сборка, type check, lint или тесты запущены, если они есть в проекте.
- Нет backend-зависимости, если она не утверждена отдельно.
- Нет post-MVP функций в первом релизе.

## Не считать блокером

- Нет аккаунтов.
- Нет родительского кабинета.
- Нет PWA или offline mode.
- Нет аудио или озвучки.
- Нет оплаты.
- Нет streak.
- Нет сложной аналитики.
- Нет уровней выше A1.
