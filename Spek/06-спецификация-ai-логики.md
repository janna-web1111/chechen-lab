## 2. Роль AI-логики в проекте

AI-логика может помогать пользователю учиться эффективнее, но не должна заменять проверенный учебный контент.

Главный принцип: чеченский учебный материал должен быть проверен человеком, особенно слова, переводы, произношение и объяснения.

## 3. Возможные функции после MVP

### Рекомендации для повторения

Система может определять, какие слова пользователю стоит повторить.

Основания:

- ошибки в квизах;
- давно не повторявшиеся карточки;
- низкий результат по теме;
- частые неправильные ответы.

### Персональный маршрут

Система может предлагать следующий шаг обучения.

Примеры:

- повторить тему;
- перейти к следующей теме;
- повторить только сложные карточки;
- пройти короткий проверочный квиз.

### Подсказки

Система может давать короткие подсказки по словам и фразам.

Ограничение: подсказки не должны противоречить проверенному контенту.

### Генерация черновиков учебного контента

AI может помогать создавать черновики карточек, примеров или квизов.

Ограничение: весь такой контент должен проходить ручную проверку перед публикацией.

## 4. Что AI не должен делать

AI не должен:

- публиковать чеченский учебный контент без проверки;
- исправлять произношение без надежной языковой базы;
- давать непроверенные грамматические объяснения как факт;
- собирать лишние персональные данные;
- принимать решения, которые пользователь не может понять или изменить.

## 5. MVP-решение

В MVP AI-логика может быть полностью исключена.

Если понадобится минимальная автоматизация, она должна быть простой и прозрачной:

- повторять слова с ошибками;
- предлагать продолжить текущую тему;
- показывать карточки из незавершенных тем.

## 6. Вопросы для уточнения

- Нужен ли AI уже в первой версии?
- Нужно ли использовать AI только для внутренней подготовки контента?
- Кто будет утверждать сгенерированные материалы?
- Должна ли система объяснять пользователю, почему предлагает повторить слово?
- Какие данные нельзя передавать внешним AI-сервисам?
