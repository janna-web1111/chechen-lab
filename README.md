# Chechen Lab

Chechen Lab - MVP веб-приложения для изучения чеченского языка с уровня A0 до A1.

Главная цель MVP: проверить, хочет ли пользователь продолжать учить чеченский после первых 10 минут.

## Главные папки

- [дорожная-карта/](дорожная-карта/) - хронологическая дорожная карта завершенных изменений.
- [планы-разработки/](планы-разработки/) - активные и завершенные планы разработки.
- [спецификации/](спецификации/) - спецификации проекта.
- [спецификации/README.md](спецификации/README.md) - карта документов по порядку: Global Spec -> Functional Map -> Feature Specs -> Technical Specs -> Visual Rules / Skills -> User Stories.
- [app-next/](app-next/) - основное Next.js приложение.
- [app/](app/) - старый статический прототип, используется как reference.
- [agents/Skills/](agents/Skills/) - инструкции для AI-агентов проекта.

## Механизм планов разработки

Рабочие планы лежат в:

[планы-разработки/](планы-разработки/)

Внутри есть две основные папки:

- [active/](планы-разработки/active/) - планы, которые еще не разработаны или не завершены.
- [completed/](планы-разработки/completed/) - завершенные планы, которые уже разработаны, протестированы и сохранены в историю.

## Правило работы

1. Пользователь просит создать план разработки.
2. План создается в `планы-разработки/active/`.
3. Разработка идет по этому плану.
4. Пока работа не закончена, план остается в `active/`.
5. Когда пользователь говорит, что разработка и тестирование закончены и просит сделать push, план сначала переносится в `completed/`.
6. После переноса выполняются проверки, commit, push и pull request.

Главное правило проекта:

```text
spec -> plan -> code
```

Если чего-то нет в спеках или плане, это не добавляется в MVP без отдельного решения.
