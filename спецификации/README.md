## Статусы документов

- `MVP` - использовать для текущей разработки.
- `mixed` - внутри есть MVP-часть и post-MVP часть; читать пометки в начале документа.
- `post-MVP` - не реализовывать в первой разработке.
- `reference` - старый или вспомогательный документ, не главный источник истины.

## Правило работы

Сначала spec -> потом plan -> потом code.

- Нет в spec -> не делаем.
- Неясно в spec -> уточняем.
- Противоречит spec -> останавливаемся.
