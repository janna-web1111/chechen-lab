# Технические

## Назначение документа

Этот документ служит коротким входом в техническую часть спецификаций проекта.

Главные подробные источники:

- `08-техническая-спецификация.md` - архитектура, frontend, backend, хранение и безопасность.
- `04-спецификация-данных.md` - сущности, поля и правила данных.
- `06-спецификация-ai-логики.md` - AI-логика и ограничения.
- `15-спецификация-мультиязычности.md` - будущая локализация интерфейса и контента.
- `16-план-разработки-mvp.md` - порядок разработки MVP.

## Техническое ядро MVP

Для первой версии рекомендуется:

- онлайн веб-приложение;
- React или Next.js;
- статичный JSON или типизированные данные для контента;
- `localStorage` для прогресса;
- русский интерфейс;
- разделение учебного контента, UI-строк и логики прогресса;
- отсутствие backend, аккаунтов и серверной базы в MVP.

## Отложенные технические решения

После проверки MVP можно вернуться к:

- backend;
- синхронизации прогресса между устройствами;
- личным кабинетам;
- PWA;
- мобильному приложению;
- офлайн-режиму;
- аудио;
- аналитике;
- админ-инструментам для контента.
