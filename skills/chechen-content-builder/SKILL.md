# Chechen Content Builder

## Назначение

Помогать готовить учебный контент для MVP, не выдавая AI-сгенерированный чеченский текст за финальный. Skill поддерживает планирование тем, структуру карточек, дизайн квизов, JSON-ready контент и чеклисты проверки для пути A0 -> A1.

## Что читать сначала

- Прочитать `references/content-rules.md` перед созданием или проверкой контента.
- Прочитать `Spek/04-спецификация-данных.md` для сущностей и полей.
- Прочитать `Spek/03-функциональная-спецификация.md` для карточек, квизов, прогресса и повторения.
- Прочитать `Spek/09-спецификация-mvp.md` перед добавлением чего-либо сверх MVP.
- Читать `Spek/15-спецификация-мультиязычности.md` только когда нужна мультиязычная структура.

## Рабочий процесс контента

1. Подтвердить, что целевой уровень A0 -> A1, если пользователь явно не указал другое.
2. Создать или проверить ровно нужную единицу контента: тему, урок, список карточек, квиз или JSON-структуру.
3. Держать тему небольшой: обычно 8-12 карточек и 5-10 вопросов квиза.
4. Включать поля, которые приложение сможет использовать: ID, порядок, чеченский текст, русский перевод, опциональная транскрипция или подсказка чтения, тема и статус проверки.
5. Помечать чеченский материал как `draft` или `needs_native_review`, если пользователь не дал уже проверенный контент.
6. Добавлять чеклист для проверяющего вместо заявления о языковой точности.

## Правила качества

- Не выдавать придуманное AI чеченское написание, произношение или грамматику как авторитетный факт.
- Не публиковать AI-сгенерированный чеченский контент без статуса проверки.
- Делать объяснения короткими; MVP не должен перегружать грамматикой.
- Предпочитать практичные темы для новичков: приветствия, семья, дом, еда, числа, простые фразы.
- Держать русский как язык интерфейса и перевода в MVP, если задача не про локализацию.
- Разделять UI-строки и учебный контент.

## Правила квиза

- Для MVP достаточно простых вопросов с вариантами ответа.
- По возможности связывать каждый вопрос с карточкой или уроком.
- Использовать один правильный ответ и 3 правдоподобных неправильных, если в теме достаточно карточек.
- Не допускать одинаковых вариантов ответа.
- Хранить порог прохождения как поле данных; если нет другого правила, использовать 70 процентов.

## Формат ответа

Для нового контента вернуть:

- Метаданные темы.
- Таблицу карточек или JSON-ready список.
- Вопросы квиза.
- Чеклист проверки носителем.
- Открытые вопросы, которые блокируют публикацию.

Для ревью вернуть:

- Сначала блокирующие проблемы.
- Проблемы структуры данных.
- Проблемы учебного качества.
- Минимальные исправления.
