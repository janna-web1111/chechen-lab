## 2. Цель MVP

Создать первую рабочую онлайн-версию приложения, которая проверяет, хочет ли пользователь продолжать изучать чеченский язык после первых 10 минут.

MVP должен дать пользователю:

- понятный старт без регистрации;
- путь A0 -> A1;
- 5 первых тем;
- карточки для изучения слов;
- квиз после темы;
- сохранение прогресса;
- простой возврат к изученным словам.

## 3. Что входит в первую разработку

В первую разработку входит только обязательное ядро MVP:

| Блок | Входит | Комментарий |
|------|:-----:|-------------|
| Онлайн веб-приложение | да | Первая платформа проекта |
| Русский интерфейс | да | Первый язык MVP |
| Путь A0 -> A1 | да | Основной учебный маршрут |
| 5 тем | да | Темы утверждены в [18-контент-mvp-5-тем.md](18-контент-mvp-5-тем.md) |
| Карточки | да | Основной учебный модуль |
| Квиз | да | Проверка после темы |
| Прогресс | да | Минимально через локальное хранение |
| Повторение | да | Простое повторение изученных слов |
| Личный кабинет | нет | После проверки MVP |
| Родительский кабинет | нет | После проверки MVP |
| Аудио | нет | После MVP |
| PWA / mobile | нет | После MVP |
| Админка | нет | После MVP |

## 4. Предварительные решения перед кодом

Перед началом разработки нужно закрыть следующие вопросы.

### Контент

- Утвердить 5 первых тем.
- Для каждой темы подготовить слова или короткие фразы.
- Для каждого слова подготовить русский перевод.
- Решить, нужна ли транскрипция в MVP.
- Проверить чеченский контент носителем или ответственным проверяющим.

### Продукт

- Утвердить порядок тем.
- Решить, сколько карточек в одной теме.
- Решить, сколько вопросов в одном квизе.
- Решить, можно ли проходить квиз повторно без ограничений.
- Решить, что считается завершением темы.

### Техника

- Выбрать frontend-стек.
- Выбрать формат хранения контента.
- Зафиксировать хранение прогресса для MVP.
- Решить, нужна ли аналитика в первой версии.
- Зафиксировать минимальные поддерживаемые экраны.

## 5. Рекомендуемые решения для MVP

Если нет дополнительных ограничений, для первой версии рекомендуется:

- frontend: React или Next.js;
- backend: не делать в MVP;
- контент: статичные JSON-файлы внутри проекта;
- прогресс: `localStorage`;
- регистрация: не делать;
- аналитика: отложить или сделать только минимальную ручную проверку;
- язык интерфейса: русский;
- чеченский контент: хранить отдельно от UI-строк;
- квиз: 5-10 вопросов на тему;
- карточки: 8-12 карточек на тему;
- повторение: список изученных карточек из завершенных тем.

## 6. Этапы разработки

### Этап 0. Подготовка контента

Результат этапа:

- утверждены 5 тем;
- подготовлены карточки;
- подготовлены вопросы квиза;
- чеченский контент проверен;
- контент записан в структурированном формате.

Критерий готовности:

- можно открыть файл контента и увидеть все темы, карточки, переводы и квизы.

### Этап 1. Структура приложения

Задачи:

- создать проект веб-приложения;
- настроить базовую структуру папок;
- создать файл контента;
- создать файл UI-текстов;
- настроить маршрутизацию или переключение экранов;
- подготовить базовый layout.

Критерий готовности:

- приложение открывается в браузере;
- виден стартовый экран;
- можно перейти к экрану пути обучения.

### Этап 2. Экран пути A0 -> A1

Задачи:

- показать список 5 тем;
- показать статус каждой темы;
- выделить текущую тему;
- показать общий прогресс;
- добавить переход в выбранную тему.

Критерий готовности:

- пользователь видит маршрут обучения и понимает, с чего начать.

### Этап 3. Экран темы

Задачи:

- показать название темы;
- показать короткое описание;
- показать количество карточек;
- показать доступность квиза;
- добавить кнопку начала карточек;
- добавить кнопку запуска квиза после карточек.

Критерий готовности:

- пользователь может открыть тему и начать изучение.

### Этап 4. Карточки

Задачи:

- показывать чеченское слово или фразу;
- показывать перевод;
- показывать транскрипцию или подсказку, если она есть;
- дать переход вперед и назад;
- отметить карточку как изученную;
- сохранить состояние в прогрессе.

Критерий готовности:

- пользователь может пройти все карточки темы.

### Этап 5. Квиз

Задачи:

- показывать вопрос;
- показывать варианты ответа;
- проверять выбранный ответ;
- показывать обратную связь;
- считать результат;
- показывать экран результата;
- сохранять результат в прогрессе.

Критерий готовности:

- пользователь может пройти квиз и увидеть процент правильных ответов.

### Этап 6. Прогресс

Задачи:

- сохранять завершенные темы;
- сохранять изученные карточки;
- сохранять результаты квизов;
- сохранять дату последнего занятия;
- восстанавливать состояние после перезагрузки страницы.

Критерий готовности:

- после закрытия и повторного открытия браузера прогресс не теряется.

### Этап 7. Повторение

Задачи:

- показать изученные карточки;
- дать фильтр по теме;
- дать повторный просмотр карточек;
- показать статус повторения.

Критерий готовности:

- пользователь может вернуться к уже изученным словам.

### Этап 8. Финальная проверка MVP

Задачи:

- проверить первый вход;
- проверить прохождение первой темы;
- проверить прохождение квиза;
- проверить сохранение прогресса;
- проверить мобильный размер экрана;
- проверить пустой прогресс нового пользователя;
- проверить повторное прохождение квиза;
- проверить отсутствие технических ошибок в основном пути.

Критерий готовности:

- пользователь может пройти первый путь без объяснений разработчика.

## 8. Что не делать до первой проверки

До первой проверки MVP не делать:

- регистрацию;
- личный кабинет;
- родительский кабинет;
- админку;
- оплату;
- аудио;
- PWA;
- мобильное приложение;
- офлайн-режим;
- сложную аналитику;
- систему наград;
- расширенные уровни A2/B1/B2;
- сложную адаптивную AI-логику.

Эти функции могут быть полезны позже, но сейчас они мешают проверить основную гипотезу.

## 9. Риски разработки

| Риск | Что может случиться | Как снизить |
|------|---------------------|-------------|
| Нет проверенного контента | Нельзя уверенно запускать MVP | Проверить слова до разработки |
| Слишком большой MVP | Разработка затянется | Держать только карточки, квиз и прогресс |
| Неясный стек | Потеря времени на архитектуру | Выбрать простой frontend без backend |
| Непонятный интерфейс | Пользователь не начнет обучение | Тестировать первый вход вручную |
| Прогресс не сохраняется | Пользователь теряет результат | Проверить `localStorage` на всех ключевых шагах |

## 10. Следующий конкретный шаг

Следующий шаг перед кодом:

1. Утвердить 5 первых тем MVP.
2. Для каждой темы подготовить 8-12 карточек.
3. Для каждой темы подготовить 5-10 вопросов квиза.
4. Проверить чеченский текст.
5. После этого создавать проект приложения.
