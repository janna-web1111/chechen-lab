# Спецификация MVP

## 2. Главная цель MVP

Главная цель MVP: проверить, хотят ли пользователи продолжать изучать чеченский язык после первых 10 минут использования приложения.

MVP должен проверить:

- понятен ли первый вход;
- интересны ли карточки и квизы;
- достаточно ли прост путь A0 -> A1;
- возвращается ли пользователь;
- нужен ли продукт родителям и детям;
- какие языки интерфейса важнее добавлять первыми.

## 3. Объем MVP

В MVP входит:

- путь A0 -> A1;
- 5 первых тем;
- короткие уроки;
- карточки для запоминания;
- квизы после тем;
- аудио-кнопка;
- сохранение прогресса;
- мобильный интерфейс;
- русский язык интерфейса.

## 4. Что не входит в MVP

В MVP не входит:

- личный кабинет;
- родительский кабинет;
- PWA;
- офлайн-режим;
- streak;
- уровни выше A1;
- сложная аналитика;
- платная подписка;
- поддержка всех будущих языков интерфейса;
- полноценная админ-панель.

## 5. Критерии готовности MVP

MVP считается готовым, если:

- пользователь может начать обучение без сложной регистрации;
- доступны 5 первых тем;
- каждая тема содержит карточки и квиз;
- аудио запускается для ключевых слов или фраз;
- прогресс сохраняется;
- пользователь видит, что уже пройдено;
- интерфейс удобно использовать на мобильном экране;
- первый путь A0 -> A1 можно пройти без технических ошибок.

## 6. Метрики MVP

Основные метрики:

- пользователь проходит первый урок;
- пользователь завершает первую тему;
- пользователь проходит первый квиз;
- пользователь возвращается на следующий день;
- средняя сессия длится 10-15 минут;
- результат квиза достигает 70% или выше.

## 7. Главные риски MVP

Риски:

- пользователю непонятно, с чего начать;
- детям становится скучно после первых экранов;
- аудио недостаточно качественное;
- чеченский контент требует доработки;
- слишком много функций отвлекают от проверки главной идеи;
- прогресс сохраняется ненадежно.
