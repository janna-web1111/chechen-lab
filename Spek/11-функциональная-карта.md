## 2. Карта навигации

```text
Header
  - название приложения
  - приветствие
  - выбор языка

Главный экран
  - карточки
  - квиз
  - сопоставление
  - правописание
  - чтение
  - охота за словами

Карточки / квиз / сопоставление / правописание / охота за словами
  -> выбор уровня
  -> активный модуль
  -> результат

Чтение
  -> список историй
  -> читалка истории
```

## 3. Экраны приложения

| ID | Экран | Назначение |
|----|-------|------------|
| `dashboard-view` | Главный экран | Показывает доступные активности |
| `level-selection` | Выбор уровня | Выбор A0, A1, A2, B1 или B2 |
| `flashcards-view` | Карточки | Изучение слов через переворот карточки |
| `quiz-view` | Квиз | Проверка знания слов через варианты ответа |
| `matching-view` | Сопоставление | Поиск пар переводов |
| `spelling-view` | Правописание | Ввод чеченского слова вручную |
| `wordhunt-view` | Охота за словами | Выбор перевода для чеченского слова |
| `reading-list-view` | Список историй | Выбор истории для чтения |
| `story-reader-view` | Читалка | История, перевод и словарь |

## 4. Потоки навигации

### Карточки

```text
Главный экран -> Карточки -> Выбор уровня -> Просмотр карточек -> Квиз
```

Пользователь должен иметь возможность вернуться:

- из просмотра карточек к выбору уровня;
- из выбора уровня на главный экран;
- из квиза на главный экран или к повторению материала.

### Чтение

```text
Главный экран -> Чтение -> Список историй -> Читалка истории
```

Пользователь должен иметь возможность:

- вернуться из истории к списку;
- вернуться из списка на главный экран;
- открыть перевод и словарь без потери позиции в истории.

### Игровые модули

```text
Главный экран -> Модуль -> Выбор уровня -> Игра -> Результат
```

К игровым модулям относятся:

- квиз;
- сопоставление;
- правописание;
- охота за словами.

## 6. Состояние приложения

Минимальное состояние, которое нужно предусмотреть:

```text
currentLevel       - текущий уровень: A0, A1, A2, B1, B2
currentWordIndex   - индекс текущего слова
isFlipped          - перевернута ли карточка
currentQuizIndex   - индекс текущего вопроса квиза
score              - текущий счет
quizPool           - набор слов для текущей сессии квиза
currentActivity    - текущий модуль
currentStoryIndex  - текущая история
currentLang        - язык интерфейса
```

## 7. Модель данных для словаря

```text
VocabularyItem:
  de: string
  ru: string
  en: string
  fr: string
  nl: string
  no: string
  tr: string
  ar: string
  ce: string
  pro: string
```

`ce` хранит чеченское слово, `pro` хранит транскрипцию.

## 8. Модель данных для истории

```text
Story:
  id: number
  level: string
  title: string
  body: StoryToken[]
  translation: object
  vocab: VocabularyItem[]

StoryToken:
  word?: string
  tip?: string
  text?: string
```

Тело истории должно поддерживать слова с подсказками и обычный текст/пунктуацию.

## 9. Хранение данных

| Данные | Где хранятся в MVP | Персистентность |
|--------|--------------------|-----------------|
| Словарь | статичный файл или база контента | статично |
| Истории | статичный файл или база контента | статично |
| UI-строки | файл локализации | статично |
| Подсказки историй | файл локализации | статично |
| Выбор языка | `localStorage` или профиль | между сессиями |
| Прогресс | локально или backend | между сессиями |
| Очки текущей игры | состояние приложения | текущая сессия |
