# Roadmap и история изменений

Эта папка хранит хронологическую историю того, что реально было сделано в проекте после завершения планов разработки.

Это не папка активных планов. Активные планы лежат в [../планы-разработки/active/](../планы-разработки/active/).

## Зачем нужна эта папка

После каждого завершенного плана нужно обновлять roadmap по областям проекта:

- что изменилось в данных;
- что изменилось во frontend;
- что изменилось в дизайне;
- что изменилось в AI-логике или AI-инструкциях;
- какие общие проектные решения были приняты.

Так сохраняется история развития проекта, а не только список задач.

## Файлы

- [хронология.md](хронология.md) - общий журнал завершенных планов.
- [области/данные.md](области/данные.md) - изменения в данных, структуре контента, `localStorage`, форматах.
- [области/ии-анализ.md](области/ии-анализ.md) - изменения AI-логики, AI-инструкций, skills и анализа.
- [области/фронтенд-и-дизайн.md](области/фронтенд-и-дизайн.md) - изменения во frontend, интерфейсе, визуальных правилах и UX.
- [области/продукт-и-спецификации.md](области/продукт-и-спецификации.md) - изменения в продуктовой логике, MVP-рамках, пользовательских историях и specs.
- [области/языковой-контент.md](области/языковой-контент.md) - изменения учебного контента и статусов проверки.

## Правило обновления

Когда пользователь говорит, что план завершен, протестирован и можно делать push:

1. Перенести файл плана из `планы-разработки/active/` в `планы-разработки/completed/`.
2. Обновить [хронология.md](хронология.md).
3. Обновить все roadmap-файлы по затронутым областям.
4. Запустить проверки.
5. Сделать commit, push и pull request.

## Формат записи

Каждая запись добавляется сверху, чтобы новые изменения были выше старых.

```text
## YYYY-MM-DD - Название завершенного плана

- План: ../планы-разработки/completed/YYYY-MM-DD-name.md
- Область: данные / AI / frontend и дизайн / продукт и спецификации / языковой контент
- Что сделано:
  - ...
- Проверки:
  - ...
- Статус: completed
```

## Главное правило

Roadmap обновляется только после завершения и тестирования плана. Идеи на будущее остаются в спецификациях или active-планах.
