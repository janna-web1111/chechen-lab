## 2. Основные сущности

### Пользователь

В MVP пользователь может существовать без полноценной учетной записи.

Возможные данные:

- идентификатор пользователя или устройства;
- выбранный язык интерфейса;
- текущий уровень;
- текущая тема;
- дата первого входа;
- дата последнего занятия.

### Уровень

Уровень описывает этап изучения языка.

Поля:

- идентификатор уровня;
- название уровня;
- порядок;
- описание;
- статус доступности.

Для MVP основной уровень: A0 -> A1.

### Тема

Тема объединяет уроки, карточки и квиз.

Поля:

- идентификатор темы;
- название;
- описание;
- уровень;
- порядок;
- список уроков;
- список карточек;
- квиз;
- статус публикации.

### Урок

Урок содержит короткий учебный блок.

Поля:

- идентификатор урока;
- тема;
- название;
- порядок;
- учебный текст;
- список карточек;
- статус публикации.

### Карточка

Карточка хранит единицу учебного материала.

Поля:

- идентификатор карточки;
- чеченское слово или фраза;
- перевод;
- транскрипция или подсказка;
- тема;
- уровень сложности;
- порядок.

### Квиз

Квиз проверяет материал темы или урока.

Поля:

- идентификатор квиза;
- тема;
- список вопросов;
- минимальный результат прохождения;
- порядок.

### Вопрос квиза

Поля:

- идентификатор вопроса;
- текст вопроса;
- варианты ответа;
- правильный ответ;
- объяснение после ответа;
- связанная карточка или урок.

### Прогресс

Прогресс показывает движение пользователя по обучению.

Поля:

- пользователь или устройство;
- завершенные темы;
- завершенные уроки;
- изученные карточки;
- результаты квизов;
- слова для повторения;
- дата последнего обновления.

## 3. Данные, которые нужно защищать

В MVP желательно не собирать лишние персональные данные.

Если личные кабинеты не входят в MVP, не нужно хранить:

- имя ребенка;
- возраст ребенка;
- email;
- телефон;
- семейные данные;
- точное местоположение.

Если эти данные появятся позже, потребуется отдельное описание приватности и правил хранения.

## 4. Правила качества контента

Учебный чеченский контент должен проходить проверку.

Для каждой карточки желательно фиксировать:

- кто добавил материал;
- кто проверил материал;
- статус проверки;
- версию перевода;
- наличие транскрипции или подсказки по чтению.

## 5. Вопросы для уточнения

- Нужно ли хранить прогресс только локально на устройстве?
- Будет ли в MVP серверная база данных?
- Кто будет проверять чеченский контент?
- Нужна ли версия контента для исправления ошибок?
- Нужно ли хранить несколько переводов одной карточки для разных языков интерфейса?
