# План 06. MVP visual polish

## Статус

- Статус: `completed`
- Ветка: `plan-mvp-visual-polish`
- Приложение: `app-next/`
- Цель: улучшить визуальное качество текущего MVP-пути без расширения продукта и без изменения учебной логики.

## Зачем этот план

После native review и MVP-candidate check основной путь работает и контент имеет статус `reviewed`. Теперь можно сделать интерфейс спокойнее, аккуратнее и понятнее для первого реального показа.

Фокус остается на первом 10-15 минутном опыте:

`start -> topics -> first topic -> cards -> quiz -> result -> saved progress -> review`.

## MVP-рамки

Входит:

- визуальный polish стартового экрана;
- улучшение карточек тем;
- улучшение экрана темы;
- улучшение карточек обучения;
- улучшение квиза, результата и повторения;
- проверка mobile/desktop layout;
- сохранение русского интерфейса и текущего MVP-потока.

Не входит:

- маркетинговый лендинг вместо приложения;
- backend;
- login или аккаунты;
- PWA/offline mode;
- аудио;
- streak;
- оплата;
- аналитика;
- новые темы;
- уровни выше A1;
- сложные анимации или брендбук.

## P0-задачи

### P0.1. Проверить текущий UI

- Статус: `done`

Проверить:

- стартовый экран;
- путь тем;
- экран темы;
- карточки;
- квиз;
- результат;
- повторение;
- mobile и desktop размеры.

Критерий готовности:

- понятен список точечных визуальных улучшений.

### P0.2. Сделать MVP-polish

- Статус: `done`

Улучшить:

- визуальную иерархию первого экрана;
- плотность и читаемость карточек тем;
- status badges и прогресс;
- внешний вид учебной карточки;
- состояние квиза и результата;
- повторение изученных карточек.

Критерий готовности:

- интерфейс выглядит цельнее, но не меняет продуктовую логику.

### P0.3. Проверить адаптивность

- Статус: `done`

Проверить:

- нет перекрытий;
- текст помещается;
- кнопки доступны;
- карточки и сетки не ломаются на мобильном размере.

Критерий готовности:

- mobile и desktop layout пригодны для ручного показа.

### P0.4. Финальные проверки

- Статус: `done`

Запустить:

```bash
npm run lint
npm run build
```

Критерий готовности:

- проверки проходят;
- план можно переносить в `completed`, коммитить, пушить и открывать PR.

## Критерий завершения плана

План можно завершить, когда:

- visual polish сделан без расширения MVP;
- основной поток остался рабочим;
- mobile/desktop layout проверен;
- `npm run lint` проходит;
- `npm run build` проходит;
- roadmap обновлен;
- план перенесен в `completed`;
- branch запушен и создан PR в `main`.
