## Зачем этот план

Первый MVP-поток технически готов к ручному тестированию, но учебный чеченский контент еще нельзя считать опубликованным: в карточках используется `TBD`, а статус материалов `needs_native_review`.

Этот план нужен, чтобы подготовить понятную основу для носителя языка или ответственного проверяющего и безопасно перенести черновой материал в приложение без публикации как проверенного контента.

## MVP-рамки

Входит:

- 5 первых тем: `greetings`, `family`, `home`, `food`, `numbers`;
- 8 карточек на тему;
- русский перевод;
- чеченское слово или короткая фраза после проверки;
- подсказка чтения после проверки;
- статус проверки `needs_native_review`, `reviewed` или `published`;
- проверка, что контент подходит для A0 -> A1;
- последующее обновление `app-next/src/data/content.ts` черновым материалом со статусом `needs_native_review`.

Не входит:

- добавление новых тем сверх 5 MVP-тем;
- уровни выше A1;
- аудио;
- backend;
- аккаунты;
- AI-генерация как финальный источник чеченского текста;
- публикация непроверенного чеченского контента.

## P0-задачи

### P0.1. Подготовить таблицу проверки

Действия:

- собрать все 40 карточек из 5 тем;
- оставить чеченский текст и подсказку чтения как `TBD`;
- добавить поля для проверяющего и комментария;
- явно указать статус `needs_native_review`.

Критерий готовности:

- проверяющий может заполнить таблицу без чтения кода приложения.

### P0.2. Проверить каждую карточку

Проверить:

- чеченское написание;
- русский перевод;
- подсказку чтения;
- соответствие уровню A0 -> A1;
- отсутствие сложной грамматики;
- отсутствие нежелательных дублей внутри темы;
- понятность для новичка.

Критерий готовности:

- у каждой карточки есть решение: оставить, исправить или заменить.

### P0.3. Обновить статусы контента

Правила:

- `needs_native_review` - если карточка не проверена;
- `reviewed` - если карточка проверена ответственным проверяющим;
- `published` - только если материал утвержден для показа пользователям.

Критерий готовности:

- статус каждой карточки соответствует реальному состоянию проверки.

### P0.4. Подготовить квизы к проверенному контенту

Действия:

- проверить, что вопросы используют реальные карточки;
- проверить, что варианты ответов не повторяются;
- проверить, что правильный ответ однозначен;
- оставить 5 вопросов на тему.

Критерий готовности:

- квизы работают с проверенными карточками и не вводят пользователя в заблуждение.

### P0.5. Перенести проверенный контент в приложение

Действия:

- обновить `app-next/src/data/content.ts`;
- не менять UI и продуктовую логику без отдельной причины;
- сохранить структуру статичных данных;
- не добавлять backend или генерацию контента.

Критерий готовности:

- в приложении карточки больше не показывают `TBD` для проверенных материалов.

### P0.6. Проверить приложение

Запустить в `app-next/`:

```bash
npm run lint
npm run build
```

Ручной сценарий:

1. открыть стартовый экран;
2. перейти к темам;
3. открыть первую тему;
4. пройти карточки;
5. пройти квиз;
6. проверить результат;
7. открыть повторение;
8. убедиться, что проверенные карточки отображаются без `TBD`.
