# План 04. Native review MVP-контента

## Статус

- Статус: `active`
- Ветка: `plan-native-review-content`
- Приложение: `app-next/`
- Цель: подготовить и провести проверку чернового чеченского контента для 5 MVP-тем, чтобы после проверки безопасно обновить статусы и исправления.

## Зачем этот план

В MVP уже внесены чеченские слова и короткие фразы для первых 5 тем, но весь материал остается `needs_native_review`.

Этот план нужен, чтобы передать проверяющему понятный список карточек, получить решения по каждой карточке и не публиковать непроверенный чеченский текст как финальный учебный материал.

## Использованные источники

- Chechnya Info, русско-чеченский разговорник: `https://chechnyainfo.com/russko-chechenskiy-razgovornik.html`
- Govzalla, переведенные слова и фразы, страница 32: `https://govzalla.com/dosh/translated?page=32`
- А. Г. Мациев, чеченско-русский словарь: `https://ps95.ru/wp-content/uploads/2018/07/Maciev_A.G_Chechensko-russkiy_slovar.pdf`

Источники используются только для подготовки черновика. Они не переводят материал в `reviewed` или `published` без проверки носителем языка или ответственным проверяющим.

## MVP-рамки

Входит:

- 5 первых тем: `greetings`, `family`, `home`, `food`, `numbers`;
- 40 текущих карточек;
- проверка чеченского написания;
- проверка русского перевода;
- решение по подсказке чтения: добавить, оставить пустой, исправить позже;
- решение по статусу: оставить `needs_native_review`, перевести в `reviewed` или `published`;
- фиксация комментариев проверяющего;
- последующее обновление `app-next/src/data/content.ts` только по подтвержденным правкам.

Не входит:

- добавление новых тем сверх 5 MVP-тем;
- уровни выше A1;
- аудио;
- backend;
- аккаунты;
- AI-генерация как финальный источник чеченского текста;
- публикация спорного или непроверенного материала.

## P0-задачи

### P0.1. Подготовить таблицу для проверяющего

- Статус: `done`
- Артефакты:
  - `планы-разработки/active/04-native-review-mvp-контента-handoff.md`
  - `планы-разработки/active/04-native-review-mvp-контента-table.tsv`
  - `планы-разработки/active/04-native-review-message-to-reviewer.md`

- собрать 40 текущих карточек;
- указать тему, ID, русский перевод и чеченский текст;
- оставить поля для решения, исправления, подсказки чтения и комментария.

Критерий готовности:

- проверяющий может работать с таблицей без чтения кода.

### P0.2. Получить проверку

- Статус: `blocked_by_reviewer`
- Non-native prefill: `done`

Я заполнил `decision`, `new_status` и `reviewer_comment` как предварительный non-native prefill:

- `decision`: `defer`;
- `new_status`: `needs_native_review`;
- `reviewer_comment`: источник, риск или причина, почему нужен проверяющий.

Это не является native review и не переводит материал в `reviewed` или `published`.

Проверить для каждой карточки:

- чеченское написание;
- русский перевод;
- уместность для A0 -> A1;
- отсутствие слишком сложной грамматики;
- нужна ли замена карточки;
- нужна ли подсказка чтения;
- можно ли менять статус на `reviewed` или `published`.

Критерий готовности:

- у каждой карточки есть одно из решений: `ok`, `fix`, `replace`, `defer`.

### P0.3. Обновить приложение по результатам

- перенести только подтвержденные исправления в `app-next/src/data/content.ts`;
- обновить `verificationStatus` только там, где есть подтверждение;
- не менять UI и продуктовую логику без отдельной причины;
- оставить неподтвержденные карточки в `needs_native_review`.

Критерий готовности:

- данные приложения отражают реальные решения проверяющего.

### P0.4. Проверить квизы

- проверить, что варианты ответов не повторяются;
- проверить, что правильный ответ однозначен;
- проверить, что квизы не используют спорные карточки как опубликованный материал.

Критерий готовности:

- квизы соответствуют проверенным или явно помеченным черновым карточкам.

### P0.5. Проверить приложение

Запустить в `app-next/`:

```bash
npm run lint
npm run build
```

Ручной сценарий:

1. открыть стартовый экран;
2. перейти к темам;
3. открыть первую тему;
4. пройти карточки;
5. пройти квиз;
6. проверить результат;
7. открыть повторение;
8. убедиться, что статусы и подсказки соответствуют реальному уровню проверки.

## Предварительный non-native triage

Этот раздел не подтверждает языковую правильность. Он нужен, чтобы проверяющий быстрее увидел карточки с повышенным риском.

### Приоритет 1. Проверить до любого статуса выше `needs_native_review`

| Card ID | Причина риска | Что попросить у проверяющего |
|---|---|---|
| `greetings-08` | Фраза из источников встречается как вопрос "как тебя/вас зовут"; для A0 нужно решить форму обращения. | Подтвердить форму `ты` или `вы` для первой версии. |
| `family-06` | Два варианта через `/`: `КӀант / воӀ`; для A0 одна карточка с двумя вариантами может быть тяжелой. | Выбрать один основной вариант или подтвердить, что оба нужны на одной карточке. |
| `family-08` | Два варианта через `/`: `Де да / нена да`; возможна зависимость от стороны семьи. | Выбрать один вариант для MVP или разделить/заменить карточку. |
| `food-01` | Два варианта через `/`: `Кхаллар / бепиг`; для A0 лучше один основной вариант. | Выбрать основной вариант для первой карточки. |
| `home-08` | В источнике встречается запись через `1`; в черновике нормализовано к `Ӏ`. | Подтвердить правильный символ по принятой орфографии проекта. |

### Приоритет 2. Проверить учебную уместность

| Card ID | Причина риска | Что попросить у проверяющего |
|---|---|---|
| `greetings-01` - `greetings-08` | Тема заменена на более короткие source-backed фразы, но все еще требует native review. | Подтвердить, что набор подходит для первого 10-минутного опыта. |
| `home-06` | `Телевизор` выглядит как заимствование и может не давать ощущения изучения нового слова. | Подтвердить, оставить ли слово в теме A0 или заменить более полезным бытовым словом. |
| `home-07` | `Стол` выглядит как заимствование и может не давать ощущения изучения нового слова. | Подтвердить, оставить ли слово в теме A0 или заменить более полезным бытовым словом. |
| `food-04` | `Йетта шура` может быть фразой/описательным названием, а не простой однословной карточкой. | Подтвердить, подходит ли для A0 и нужен ли более базовый продукт. |

### Приоритет 3. Проверить единый стандарт записи

- Подтвердить единый стандарт символа `Ӏ` и допустимость нормализации источниковых `I`/`І`/`1` к `Ӏ`.
- Выбрать, используем ли прописные буквы для всех приветствий или обычный регистр.
- Решить, допускаются ли варианты через `/` в одной карточке MVP.
- Решить, нужны ли подсказки чтения сейчас; если нет, оставить `Ожидает проверки`.
- Для статуса `published` требовать явное утверждение проверяющего; иначе использовать максимум `reviewed`.

## Открытые вопросы

- Кто будет ответственным проверяющим чеченского контента?
- После первой проверки использовать `reviewed` или сразу `published`?
- Нужны ли подсказки чтения в MVP, если проверяющий не готов быстро их подтвердить?
- Нужно ли оставлять варианты через `/`, например `КӀант / воӀ`, или выбрать один вариант для карточки A0?

## Критерий завершения плана

План можно завершить, когда:

- таблица проверки заполнена проверяющим;
- спорные карточки исправлены, заменены или оставлены в `needs_native_review`;
- статусы карточек отражают реальный уровень проверки;
- подтвержденные изменения перенесены в `app-next/src/data/content.ts`;
- спецификация контента обновлена;
- `npm run lint` проходит;
- `npm run build` проходит;
- основной ручной поток не показывает неподтвержденный материал как опубликованный.
