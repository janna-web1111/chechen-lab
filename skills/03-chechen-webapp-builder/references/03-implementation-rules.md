# Правила реализации веб-MVP

## Форма продукта

Строить простое онлайн веб-приложение для браузера. Приложение должно позволять новому пользователю быстро начать, пройти путь A0 -> A1, изучать карточки, проходить квизы, сохранять прогресс и повторять изученные слова.

## Архитектура данных

Держать контент отдельно от UI-компонентов.

Рекомендуемая структура контента:

- `levels`
- `themes`
- `lessons`
- `cards`
- `quizzes`
- `quizQuestions`
- `uiStrings`

Рекомендуемая структура прогресса:

- ID завершенных тем
- ID завершенных уроков
- ID изученных карточек
- результаты квизов
- слова для повторения
- дата первого входа
- дата последнего занятия

## Хранение

Использовать `localStorage` для прогресса MVP, если пользователь явно не просит backend. Делать ключи с namespace, например `chechenLab.progress.v1`.

Не хранить:

- имя ребенка
- возраст
- email
- телефон
- семейные данные
- точное местоположение

## Требования к UI

Использовать короткие русские тексты интерфейса в MVP. Держать строки готовыми к локализации.

Нужные экраны:

- Стартовый экран.
- Путь обучения.
- Тема.
- Урок или карточки.
- Квиз.
- Результат.
- Повторение.

Нужные состояния:

- новый пользователь без прогресса
- восстановленный прогресс
- повторное прохождение квиза
- пустой контент
- ошибка загрузки или разбора контента

## Поведение квиза

- 5-10 вопросов на тему.
- Для MVP достаточно multiple-choice.
- Сохранять процент результата.
- Использовать 70 процентов как стандартный порог прохождения.
- Разрешать повторные попытки, если спецификация позже не изменится.

## Отложенные функции

Не реализовывать для MVP:

- login
- кабинет родителей
- backend sync
- payments
- PWA
- native mobile app
- offline mode
- audio
- streak
- advanced analytics
- advanced AI
- уровни A2/B1/B2
