# Техническая спецификация

## 1. Назначение документа

Этот документ описывает техническую реализацию проекта: архитектуру, frontend, backend, хранение данных, безопасность, аудио и требования к качеству.

## 2. Общий технический подход

MVP должен быть технически простым, чтобы быстрее проверить интерес пользователей.

Главные принципы:

- мобильный интерфейс в первую очередь;
- минимальная сложность регистрации;
- простое хранение прогресса;
- возможность добавить backend позже;
- подготовка контента к будущей мультиязычности.

## 3. Frontend

Frontend должен обеспечивать:

- отображение учебного пути;
- уроки;
- карточки;
- квизы;
- аудио;
- прогресс;
- адаптацию под мобильные экраны.

Интерфейс должен быть готов к будущей локализации.

## 4. Backend

В MVP backend может быть не нужен, если прогресс хранится локально и контент встроен в приложение.

Backend понадобится, если нужно:

- хранить прогресс между устройствами;
- создавать личные кабинеты;
- управлять контентом через админку;
- добавлять родительский кабинет;
- собирать аналитику;
- синхронизировать данные.

## 5. База данных

В MVP возможны два подхода:

- локальное хранение прогресса на устройстве;
- серверное хранение прогресса в базе данных.

Для первой версии предпочтительно минимизировать персональные данные.

## 6. Контент

Учебный контент должен быть структурирован:

- уровни;
- темы;
- уроки;
- карточки;
- квизы;
- аудиофайлы;
- переводы.

Контент должен быть отделен от интерфейсной логики, чтобы его можно было редактировать и переводить.

## 7. Аудио

Система должна поддерживать аудиофайлы для карточек и уроков.

Требования:

- быстрый запуск аудио;
- понятная кнопка воспроизведения;
- обработка ошибки загрузки;
- единый формат хранения аудио;
- связь аудиофайла с карточкой или фразой.

## 8. Безопасность и приватность

MVP не должен собирать лишние персональные данные.

Если личные кабинеты не используются, необходимо избегать хранения:

- имени ребенка;
- возраста;
- email;
- телефона;
- точного местоположения.

Если аналитика используется, она должна быть минимальной и понятной.

## 9. Производительность

Приложение должно:

- быстро открываться на мобильном устройстве;
- не тормозить при переключении карточек;
- быстро запускать аудио;
- сохранять прогресс без заметной задержки.

## 10. Вопросы для уточнения

- На какой платформе будет сделан первый MVP: web, PWA или mobile?
- Нужно ли хранить контент в JSON, базе данных или CMS?
- Нужен ли backend в первой версии?
- Где будут храниться аудиофайлы?
- Нужна ли аналитика в MVP?
