# Спецификация мультиязычности

Источник: адаптировано из найденных спецификаций `chechen-kids-app-specs/specs/feature-spec-i18n.md`.

> Статус: `post-MVP`.
>
> В MVP интерфейс только на русском языке. Этот документ хранит будущую техническую схему локализации,
> чтобы не смешивать ее с первой разработкой. Переключатель языков, переводы UI на другие языки,
> многоязычные истории и RTL-поддержка не входят в MVP.

## 1. Назначение документа

Этот документ описывает мультиязычность интерфейса, словаря и подсказок. Он дополняет:

- [04-спецификация-данных.md](../04-TechnicalSpecs/04-спецификация-данных.md);
- [07-спецификация-интерфейса.md](../05-VisualRules-Skills/07-спецификация-интерфейса.md);

## 2. Описание

Система мультиязычности должна позволять пользователю переключать язык интерфейса. Все основные UI-тексты, переводы словаря и подсказки в историях должны отображаться на выбранном языке.

## 3. Поддерживаемые языки

| Код | Язык | UI | Словарь | Истории |
|-----|------|:--:|:-------:|:-------:|
| `de` | Deutsch | да | да | да |
| `ru` | Русский | да | да | да |
| `en` | English | да | да | да |
| `fr` | Francais | да | да | да |
| `nl` | Nederlands | да | да | да |
| `no` | Norsk | да | да | да |
| `tr` | Turkce | позже | да | позже |
| `ar` | Arabic | позже | частично | позже |

## 4. Архитектура локализации

```text
LANGUAGES
  - реестр языков: код, название, флаг или label

UI
  - все строки интерфейса
  - ключ -> перевод по языкам

TIP_TR
  - подсказки для слов в историях
  - базовая подсказка -> перевод по языкам

t(key)
  - получить UI-строку для текущего языка

w(obj)
  - получить перевод словарного объекта для текущего языка

tipT(tip)
  - получить перевод подсказки для текущего языка
```

## 5. Переключение языка

Поток переключения:

```text
Пользователь выбирает язык
  -> currentLang обновляется
  -> выбор сохраняется
  -> интерфейс перерисовывается
  -> атрибут lang у документа обновляется
```

Минимальные требования:

- выбор языка должен быть доступен с главного экрана;
- выбранный язык должен сохраняться между сессиями;
- при отсутствии перевода должен быть fallback;
- fallback-язык для MVP: русский или английский;
- переключение языка не должно сбрасывать прогресс текущего упражнения.

## 6. Требования к данным

Каждое слово словаря должно поддерживать переводы:

```text
de, ru, en, fr, nl, no, tr, ar, ce, pro
```

Где:

- `ce` - чеченское слово;
- `pro` - транскрипция;
- остальные поля - переводы на языки интерфейса.

## 7. Как добавить новый язык

1. Добавить язык в реестр `LANGUAGES`.
2. Добавить перевод для каждого ключа UI.
3. Добавить перевод для каждого слова словаря.
4. Добавить перевод каждой истории.
5. Добавить переводы подсказок в историях.
6. Проверить верстку на длинных строках.
7. Для RTL-языков отдельно проверить направление текста.
