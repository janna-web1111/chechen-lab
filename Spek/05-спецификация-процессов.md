# Спецификация процессов

Этот документ описывает основные пользовательские и системные процессы приложения: как отдельные функции соединяются в понятные сценарии.

## 2. Первый вход пользователя

Цель процесса: пользователь должен быстро понять, что делать, и начать первый урок.

Шаги:

1. Пользователь открывает приложение.
2. Система показывает стартовый экран.
3. Пользователь видит путь A0 -> A1.
4. Пользователь начинает первую тему.
5. Система сохраняет факт первого входа.

Критерий успешности: пользователь доходит до первого учебного действия без сложной настройки.

## 3. Прохождение темы

Цель процесса: пользователь изучает тему и закрепляет материал.

Шаги:

1. Пользователь открывает тему.
2. Система показывает краткую структуру темы.
3. Пользователь проходит уроки или карточки.
4. Пользователь проходит квиз.
5. Система показывает результат.
6. Система сохраняет прогресс.

Критерий успешности: тема получает статус завершенной или частично завершенной.

## 4. Изучение карточки

Цель процесса: пользователь знакомится со словом или фразой.

Шаги:

1. Система показывает чеченское слово или фразу.
2. Пользователь смотрит перевод.
3. Пользователь смотрит транскрипцию или подсказку по чтению, если она есть.
4. Пользователь переходит к следующей карточке.
5. Система обновляет статус карточки.

Критерий успешности: карточка отмечается как просмотренная или изученная.

## 5. Прохождение квиза

Цель процесса: проверить понимание материала.

Шаги:

1. Пользователь запускает квиз.
2. Система показывает вопрос.
3. Пользователь выбирает ответ.
4. Система показывает правильность ответа.
5. После последнего вопроса система показывает результат.
6. Система сохраняет результат квиза.

Критерий успешности: пользователь получает результат и понимает, нужно ли повторить тему.

## 6. Возврат к обучению

Цель процесса: пользователь продолжает обучение после перерыва.

Шаги:

1. Пользователь открывает приложение повторно.
2. Система находит сохраненный прогресс.
3. Система показывает текущую тему или следующий шаг.
4. Пользователь продолжает обучение.

Критерий успешности: пользователь не теряет место, где остановился.

## 7. Повторение слов

Цель процесса: пользователь возвращается к уже изученным словам.

Шаги:

1. Пользователь открывает раздел повторения.
2. Система показывает карточки из пройденных тем.
3. Пользователь повторяет слова, переводы и транскрипцию.
4. Система обновляет статус повторения.

Критерий успешности: пользователь может повторять материал без повторного прохождения всей темы.
