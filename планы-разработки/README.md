# Папка рабочих планов

Эта папка хранит планы разработки и историю завершенных работ.

## Структура

- [active/](active/) - активные планы, которые еще не разработаны или не закончены.
- [completed/](completed/) - завершенные планы, которые уже разработаны, проверены и готовы к push.

## Правило работы с планами

1. Когда пользователь просит создать план разработки, новый файл плана создается в `active/`.
2. Пока разработка не закончена, план остается в `active/`.
3. Когда пользователь говорит, что разработка и тестирование плана закончены и просит сделать push, перед push нужно перенести файл плана из `active/` в `completed/`.
4. После переноса плана в `completed/` нужно выполнить проверки, commit, push и pull request по правилам `AGENTS.md`.

## Формат имени файла

Рекомендуемый формат:

```text
YYYY-MM-DD-короткое-название-плана.md
```

Пример:

```text
2026-06-05-content-review.md
```

## Что должно быть внутри плана

- цель;
- границы MVP;
- список задач;
- критерии готовности;
- проверки перед завершением;
- статус: `active` или `completed`.

## Главное правило

Сначала plan в `active/` -> потом разработка -> потом тестирование -> потом перенос в `completed/` -> потом push.
